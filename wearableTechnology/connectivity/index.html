Brainstorming

For this project, I will be redoing my Riverdale-based project from the senses portion of the semester. For a quick reminder, I created a skirt that emanated sound whenever the user shook their hips to represent the unnecessarily loud sounds coming from the Riverdale actors during their Heather's special.

Here is the project:

My idea for this project is to now create a project wear the user would be able to shake their hips and now play a song from a separate device, much like the actual musical episode.

Block Schematic

Process

to help me start off this project, I referenced two of the examples that Jingwen had provided in class: the Arduino BLE accelerometer code and the p5 code.

Using these examples (and by downloading all of the relevant files and libraries onto my local computer), I began the process of putting my project together.

My first challenge was figuring out how to register when the accelerometer was and was not in motion. I was able to figure this out by creating a variable to keep track of two data points; the absolute values of the position of accelerometer was in the moment and the position of the accelerometer right beforehand. I made the program recognize if those two values were far enough apart to consider the accelerometer to be "in motion", and it worked!

Unfortunately, I came across another issue; sometimes, the program would keep alternating between "motion" and "not in motion" since it kept tracking the position of the accelerometer so quickly, so I created a few more variables so that it was a little bit more forgiving of smaller, longer movements.

Finally, now that I was able to get the computer to recognize the two states of "motion" and "not in motion" accurately, I took an MP3 clip of the song "Candy Store" from Heathers: The Musical, and I was able to get it to play using the p5.js sound library.
